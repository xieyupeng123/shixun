## 提示词正文

第一、二阶段已完成（核心跑通+战斗闭环），现在进入第三阶段：系统完善。

请先阅读系统提示词（00-项目系统提示词.md），确认当前代码结构，然后按以下顺序实现：

### 任务1：UI系统（ui.py）

UI类负责所有HUD渲染：

**血条/能量条** show_bar(current, max_amount, bg_rect, color)：
- 背景色 + 填充色（按比例） + 边框
- 位置：左上角 (10, 10) 和 (10, 34)

**经验值** show_exp(exp)：
- 右下角显示数字，深色背景+边框

**武器/魔法图标** weapon_overlay / magic_overlay：
- 左下角选择框，200ms切换时金色边框高亮

**Boss血条** show_boss_health_bar(boss_sprite)：
- 顶部居中，300×18像素
- 颜色随阶段变化：phase1(深红) / phase2(淡红) / phase3(紫色)
- 显示 "当前HP / 最大HP"

**Boss阶段** show_boss_phase_hud(boss_sprite)：
- 顶部居中，显示 "BOSS: Phase 1 - Normal" 等

**无敌标记** show_invincible_status(player)：
- 顶部居中金色 "INVINCIBLE" 文字+边框

**Boss击杀进度** show_boss_status(boss_kills)：
- 右上角显示 "BOSS 2/3"

### 任务2：升级系统（upgrade.py）

**Upgrade类**：
- B键暂停时显示，覆盖半透明背景
- 五列布局（health/energy/attack/magic/speed）
- 左右键选择属性，上键升级，下键降级
- 300ms移动冷却防连触

**升级逻辑** trigger(player)：
- 检查exp >= upgrade_cost 且 stats < max_stats
- stats *= 1.2（取整）
- upgrade_cost *= 1.4（取整）

**降级逻辑** downgrade(player)：
- 检查stats > PLAYER_BASE_STATS
- refund = upgrade_cost / 1.4
- stats /= 1.2
- upgrade_cost /= 1.4
- 不低于初始值

**Item类**：
- 显示属性名、当前值条、升级费用
- 选中时高亮背景

### 任务3：存档系统（save_manager.py）

**save_game(data, filepath='savegame.json')**：
- JSON原子写入：先写临时文件，成功后os.replace原子替换
- 异常时清理临时文件

**load_game(filepath='savegame.json')**：
- 读取JSON，失败返回None

**存档数据结构**：
```json
{
    "player": {
        "pos": {"x": 1024, "y": 768},
        "health": 180, "energy": 85, "exp": 350,
        "stats": {"health": 240, "energy": 120, "attack": 18, "magic": 7, "speed": 400},
        "max_stats": {...},
        "upgrade_cost": {...},
        "weapon_index": 2, "magic_index": 0
    },
    "defeated_enemies": [{"x": 512, "y": 384}, ...],
    "destroyed_grass": [{"x": 256, "y": 128}, ...]
}
```

**集成点**：
- main.py：启动时检测savegame.json，存在则弹出对话框C继续/N新游戏
- level.py：P键暂停时调用save_game(level.get_savable_state())
- player.py：to_dict() / from_dict() 序列化/反序列化

### 任务4：世界地图（MapRenderer在level.py内）

MapRenderer.draw()：
- 按M键切换全屏覆盖
- 半透明黑色背景(0,0,0,200)
- 金色标题 "WORLD MAP (M to close)"
- 障碍网格：深灰色=障碍，浅绿色=可通行
- 敌人红点（普通敌人小圆，Boss大红叉+脉冲+BOSS标签）
- 玩家绿点（脉冲+YOU标签）
- 底部信息：坐标 + Boss击杀进度
- 右下角图例：You(绿)/Enemy(红)/Boss(红叉)
- 金色边框

### 任务5：场景切换（集成到MapManager/Level）

传送点机制：
- 玩家移动到传送点坐标附近（距离 < 半格）触发
- 淡出动画：黑色遮罩alpha从0→255
- 销毁当前关卡所有精灵
- 加载目标地图CSV
- 创建新Level实例
- 玩家状态（血量/能量/经验/属性/武器）跨地图保留
- 淡入动画：alpha从255→0
- 防抖机制：切换后短暂冷却防止连续触发

### 任务6：死亡/胜利结算界面

**死亡界面**（main.py）：
- 暗红色背景(40,0,0)
- "You Died"红色大字
- 经验统计
- 红色浮动粒子（30个，随机运动+边缘环绕）
- R重开：完全重置所有状态
- ESC退出

**胜利界面**（main.py）：
- 深色背景(10,10,30)
- "YOU WIN!"金色大字
- "The Boss has been defeated!"副标题
- 经验统计
- 金色浮动粒子
- R再玩 / ESC退出

**重启逻辑** _restart_game()：
- 停止所有音效和音乐
- 重置MusicState
- 重新创建Level
- 重启BGM

### 任务7：水面像素级检测

MapManager._mark_water_tiles()：
- 读取ground.png的Alpha通道
- 检测蓝色像素：blue > red 且 blue > green 且 blue > 60
- 标记为障碍Tile
- 边界检测：外圈2格始终为障碍
- 缓存机制：同一地图不重复检测
