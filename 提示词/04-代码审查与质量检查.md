# 代码审查提示词

## 使用方式

每个阶段完成后，将本提示词发送给AI进行代码审查。

---

## 提示词正文

请对我刚完成的代码进行全面审查，检查以下所有维度：

### 架构合规性检查

1. **分层架构**：
   - [ ] 上层是否依赖下层？（不允许下层依赖上层）
   - [ ] 同层模块之间是否通过回调函数通信？（不允许直接引用）
   - [ ] Level是否作为核心协调者？（不应把所有逻辑堆在Level里）

2. **单一职责**：
   - [ ] 每个类是否只负责一个功能域？
   - [ ] CombatManager是否独立处理战斗逻辑？
   - [ ] BossManager是否独立处理Boss生命周期？
   - [ ] MapManager是否独立处理地图加载？

3. **组合模式**：
   - [ ] Level是否通过组合集成CombatManager、BossManager、MapRenderer？
   - [ ] ResourceManager是否为单例模式？

4. **数据驱动**：
   - [ ] 武器/怪物/魔法属性是否集中在settings.py？
   - [ ] 新增类型是否只需加配置不改核心代码？

### 代码质量检查

5. **封装**：
   - [ ] 类的内部状态是否通过方法暴露？（不允许外部直接访问私有状态）
   - [ ] hitbox、pos等关键属性是否有合理的访问控制？

6. **低耦合**：
   - [ ] Player是否直接知道Enemy的存在？（应该不知道）
   - [ ] 模块之间的依赖是否尽可能少？

7. **delta time驱动**：
   - [ ] 所有移动是否使用speed × dt计算？
   - [ ] 动画是否使用frame_index += animation_speed × dt？
   - [ ] 是否有固定帧数的硬编码？（不应该有）

8. **异常降级**：
   - [ ] 图片加载失败是否有try/except降级？
   - [ ] 音效加载失败是否静默跳过？
   - [ ] 资源缺失是否导致崩溃？（不应该）

### 功能完整性检查

9. **交互操控**：
   - [ ] WASD/方向键移动
   - [ ] 鼠标左键攻击
   - [ ] 鼠标右键魔法
   - [ ] Q键切换武器（200ms冷却）
   - [ ] E键切换魔法
   - [ ] B键升级菜单
   - [ ] M键世界地图
   - [ ] P键存档
   - [ ] 空格无敌模式

10. **数据准确性**：
    - [ ] 武器属性是否与settings.py一致？
    - [ ] 怪物属性是否与settings.py一致？
    - [ ] Boss阶段倍增是否正确（×1/×2/×4）？
    - [ ] 玩家属性上限是否正确？

### 性能检查

11. **帧率**：
    - [ ] 复杂场景是否稳定60FPS？
    - [ ] 是否有不必要的每帧计算？
    - [ ] 粒子系统是否在播放完毕后自动kill？

12. **内存**：
    - [ ] 精灵是否在不需要时kill？
    - [ ] ResourceManager缓存是否合理？
    - [ ] 是否有内存泄漏风险？

### 请输出

1. 发现的问题列表（按严重程度排序：严重/中等/轻微）
2. 每个问题的具体位置（文件:行号）
3. 修复建议
4. 整体评分（1-10分）

如果代码没有问题，请确认"审查通过"并列出关键检查点的通过状态。
