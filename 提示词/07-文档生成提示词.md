# 文档生成提示词

## 使用方式

需要生成或更新项目文档时，使用本提示词。

---

## 提示词正文

请根据当前代码状态，生成/更新以下文档：

### 需求规格说明书（SRS）

基于代码实现，更新以下内容：
1. 功能需求清单（FR-01到FR-18）的实现状态
2. 非功能需求（性能/可靠性/可维护性/可扩展性）的验证结果
3. 验收标准的通过情况
4. 需求追踪矩阵的更新

### 概要设计说明书（HLD）

基于代码结构，更新以下内容：
1. 四层分层架构的模块映射
2. 面向对象类图（继承与组合关系）
3. 核心状态机设计（游戏主流程/敌人AI/Boss阶段）
4. 数据结构设计（配置/地图/存档）

### 设计与实现文档

基于代码实现，更新以下内容：
1. 系统架构设计（分层职责说明）
2. 面向对象设计（类职责详述）
3. 核心算法分析（A*/分轴碰撞/Y轴排序/FSM）
4. 关键交互时序（攻击/场景切换/怪物AI）
5. 测试方案（功能测试/边界测试）

### 代码注释

为以下文件添加/更新docstring和行内注释：
1. 所有类的docstring（职责、参数、返回值）
2. 所有公共方法的docstring
3. 复杂逻辑的行内注释
4. 关键设计决策的说明

### README.md

生成项目README，包含：
1. 项目简介
2. 技术栈
3. 运行方法
4. 操作说明
5. 项目结构
6. 开发文档链接

## 文档规范

1. **格式**：使用Markdown格式
2. **结构**：章节编号、表格、代码块
3. **内容**：基于实际代码实现，不要臆测
4. **更新**：每次功能变更后同步更新文档

## 注意事项

1. 文档必须与代码保持一致
2. 新增功能必须同步更新文档
3. 修复Bug后更新相关文档
4. 重大重构后全面更新文档
