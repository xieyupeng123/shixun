## 提示词正文

我正在开发「星露谷战斗模块」（Stardew Valley Action X），技术栈Python 3.12 + Pygame 2.6，窗口1280×720，60FPS，delta time驱动。

请先阅读我的系统提示词（00-项目系统提示词.md）和设计文档（需求规格说明书、概要设计说明书），然后按以下顺序实现第一阶段：

### 任务1：项目骨架与配置（settings.py + constants.py + support.py）

创建以下文件：

**settings.py** 包含：
- 窗口参数：WIDTH=1280, HEIGHT=720, FPS=60, TILESIZE=64
- HITBOX_OFFSET字典：player=-26, object=-40, grass=-10, invisible=0
- UI参数：BAR_HEIGHT=20, HEALTH_BAR_WIDTH=200, ENERGY_BAR_WIDTH=140, ITEM_BOX_SIZE=80
- 颜色常量：WATER_COLOR='#71ddee', UI_BG_COLOR='#222222', 等
- weapon_data字典：sword(15伤害/100ms冷却), lance(30/400ms), axe(20/300ms), rapier(8/50ms), sai(10/80ms)
- magic_data字典：flame(strength=10, cost=20), heal(strength=50, cost=10)
- monster_data字典：squid(50HP/15伤害/150速度/100经验), raccoon(300/40/100/500), spirit(40/8/200/110), bamboo(35/5/150/120)
- PLAYER_BASE_STATS：health=200, energy=100, attack=15, magic=5, speed=300
- BOSS_PHASE_DATA：阶段1(无色/×1), 阶段2(淡红/×2), 阶段3(紫色/×4)

**constants.py** 包含：
- CSV实体编码：ENTITY_PLAYER_SPAWN='394', ENTITY_BOSS_SPAWN='392', ENTITY_BAMBOO='390', ENTITY_SPIRIT='391', ENTITY_SQUID='393'
- ENTITY_MONSTER_MAP映射
- 精灵类型：SPRITE_GRASS='grass', SPRITE_OBJECT='object', SPRITE_INVISIBLE='invisible'
- 图层名：LAYER_BOUNDARY='boundary', LAYER_GRASS='grass', LAYER_OBJECT='object', LAYER_ENTITIES='entities'

**support.py** 包含：
- get_path(path)：基于__file__的相对路径解析
- import_csv_layout(path)：读取CSV返回二维列表
- import_folder(path)：读取文件夹所有PNG，返回Surface列表（排序）

### 任务2：资源管理器（resource_manager.py + sound_manager.py）

**ResourceManager**（单例模式）：
- instance()类方法获取全局实例
- get_image(path)：加载并缓存PNG，返回Surface
- get_sound(path, volume)：加载并缓存音效，返回Sound
- get_font(path, size)：加载并缓存字体，返回Font
- get_folder_images(path)：加载文件夹所有图片，返回列表
- stop_all_sounds()：停止所有已缓存音效

**SoundManager**：
- load(name, volume)：按文件名加载音效，失败返回None
- play_music(name, loops, volume)：播放背景音乐，失败静默跳过

### 任务3：Tile瓦片精灵（tile.py）

Tile继承pygame.sprite.Sprite：
- 参数：pos, groups, sprite_type, surface=None
- 根据sprite_type设置不同的hitbox偏移（使用HITBOX_OFFSET）
- object类型rect需要y方向上移一个TILESIZE

### 任务4：Entity基类（entity.py）

Entity继承pygame.sprite.Sprite，提供三个核心能力：

1. **统一移动**：`move(speed, pos, dt)` — 移动量=速度×方向×dt，方向向量归一化
2. **分轴碰撞**：`collision(direction)` — 先水平后垂直，检测obstacle_sprites碰撞并修正位置
3. **动画帧管理**：`animate(dt)` — frame_index += animation_speed × dt，支持不同帧数动画混合
4. **波形闪烁**：`wave_value()` — 基于sin函数返回255或0（用于无敌闪烁）

### 任务5：Player玩家类（player.py）

继承Entity，实现：

1. **初始化**：
   - 加载12组动画（up/down/left/right × idle/walk/attack）
   - 初始属性：health=200, energy=100, attack=15, magic=5, speed=300
   - 武器系统：weapon_index=0, weapon='sword', 200ms切换冷却
   - 魔法系统：magic_index=0, magic='heal'
   - 受伤系统：vulnerable=True, invulnerability_duration=500ms

2. **输入处理** input()：
   - WASD/方向键移动，设置direction和status
   - 鼠标左键攻击（attacking=True，回调create_attack）
   - 鼠标右键魔法（回调create_magic）
   - Q键切换武器（200ms冷却）
   - E键切换魔法
   - 空格切换无敌模式（速度×2，攻击×2，金色光效）

3. **状态管理** get_status()：
   - 无输入时添加'_idle'后缀
   - 攻击时添加'_attack'后缀
   - 例如：'down' → 'down_idle' → 'down_attack'

4. **冷却系统** cooldowns()：
   - 攻击冷却 = base_cooldown + weapon_cooldown
   - 切换冷却 200ms
   - 受伤无敌 500ms

5. **伤害计算**：
   - get_full_weapon_damage() = attack + weapon_damage（无敌时×2）
   - get_full_magic_damage() = magic + spell_damage（无敌时×2）

6. **序列化**：to_dict() / from_dict() 用于存档系统

### 任务6：主菜单与游戏入口（main.py）

Game类实现全局状态机：

1. **主菜单**（game_state='start'）：
   - 深色背景(20,20,40) + 蓝色浮动粒子
   - 金色标题正弦浮动动画：y_offset = 10 * sin(time * 0.005)
   - 菜单选项：Start Game / Quit Game
   - 上下键选择，高亮脉冲（每100ms切换），回车确认
   - ESC直接退出
   - 背景音乐循环播放

2. **游戏中**（game_state='game'）：
   - 创建Level实例，运行游戏循环
   - 检测玩家死亡 → 切换到death状态
   - 检测Boss全灭 → 切换到victory状态

3. **死亡界面**（game_state='death'）：
   - 暗红色背景(40,0,0) + 红色浮动粒子
   - "You Died" + 经验统计
   - R重开 / ESC退出

4. **胜利界面**（game_state='victory'）：
   - 深色背景(10,10,30) + 金色浮动粒子
   - "YOU WIN!" + 经验统计
   - R再玩 / ESC退出

5. **存档对话框**：
   - 启动时检测savegame.json是否存在
   - 存在时弹出居中对话框：C继续 / N新游戏

### 任务7：Level关卡协调者（level.py骨架）

Level类初始结构：
- 组合模式：self.combat = CombatManager(self), self.boss = BossManager(self), self.map_renderer = MapRenderer(self)
- Sprite组：visible_sprites (YSortCameraGroup), obstacle_sprites
- 兼容性属性：通过@property代理到combat/boss
- 主循环run(dt)：更新 → 绘制UI → Boss音乐检测 → 地图覆盖

**YSortCameraGroup**：
- 继承pygame.sprite.Group
- custom_draw(player)：以玩家为中心偏移，按Y坐标排序绘制所有精灵
- 绘制地面贴图（ground.png）作为最底层

### 任务8：地图系统（map_manager.py）

MapManager.load()流程：
1. 读取4个CSV文件：FloorBlocks/Grass/Objects/Entities
2. 遍历Boundary层 → 创建不可通行Tile
3. 遍历Grass层 → 创建可破坏草地Tile（随机选择图片）
4. 遍历Objects层 → 创建装饰物体Tile
5. 构建寻路网格（build_grid）
6. 水域检测：读取ground.png像素蓝色通道，标记水域为障碍
7. 遍历Entities层：
   - 394 → 创建Player
   - 392 → 记录Boss出生点，spawn_boss(phase=1)
   - 390/391/393 → 创建对应怪物Enemy

### 任务9：A*寻路（pathfinding_utils.py）

实现以下函数：
- heuristic(a, b)：曼哈顿距离
- astar(grid, start, goal)：A*算法，返回路径列表
- build_grid(map_width, map_height, tile_size, obstacle_sprites)：构建0/1网格
- pos_to_grid(pos, tile_size)：像素坐标→网格坐标
- grid_to_pos(grid_coord, tile_size)：网格坐标→像素坐标（中心点）
