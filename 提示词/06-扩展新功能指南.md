# 扩展新功能指南

## 使用方式

需要添加新武器/怪物/魔法/地图时，参考本指南。

---

## 添加新武器

### 步骤1：在settings.py添加配置

```python
weapon_data = {
    # ... 已有武器 ...
    'new_weapon': {'cooldown': 200, 'damage': 25, 'graphic': new_weapon_path},
}
```

### 步骤2：准备美术资源

在 `graphics/weapons/new_weapon/` 目录下放置4个方向的图片：
- up.png
- down.png
- left.png
- right.png

### 步骤3：测试验证

1. 启动游戏，按Q键切换到新武器
2. 验证伤害计算：damage = player.attack + weapon.damage
3. 验证冷却时间正确
4. 验证无敌模式下伤害×2

**无需修改的核心代码**：weapon.py、player.py、level.py都会自动适配新配置。

---

## 添加新怪物

### 步骤1：在settings.py添加配置

```python
monster_data = {
    # ... 已有怪物 ...
    'new_monster': {
        'health': 60,
        'exp': 150,
        'damage': 20,
        'attack_type': 'slash',  # 对应粒子特效类型
        'attack_sound': slash_sound_path,
        'speed': 180,
        'resistance': 3,
        'attack_radius': 90,
        'notice_radius': 380
    },
}
```

### 步骤2：准备美术资源

在 `graphics/monsters/new_monster/` 目录下放置3个动画文件夹：
- idle/  （待机动画帧）
- move/  （移动动画帧）
- attack/（攻击动画帧）

### 步骤3：在constants.py添加CSV编码

```python
ENTITY_NEW_MONSTER = '395'  # 选择一个未使用的编码

ENTITY_MONSTER_MAP = {
    # ... 已有映射 ...
    ENTITY_NEW_MONSTER: 'new_monster',
}
```

### 步骤4：在地图CSV中放置

在 map_Entities.csv 的对应位置填入 '395'。

### 步骤5：测试验证

1. 进入对应地图，验证怪物正确生成
2. 验证AI行为：idle→追踪(A*绕障碍)→攻击
3. 验证伤害计算和经验掉落
4. 验证死亡粒子和经验球特效

**无需修改的核心代码**：enemy.py会自动读取新配置。

## 添加新魔法

### 步骤1：在settings.py添加配置

```python
magic_data = {
    # ... 已有魔法 ...
    'new_spell': {'strength': 15, 'cost': 15, 'graphic': new_spell_path, 'spell_sound': new_spell_sound_path},
}
```

### 步骤2：准备美术资源

- 粒子图片：`graphics/particles/new_spell/`
- 音效文件：`audio/new_spell.wav`

### 步骤3：在magic.py添加释放逻辑

在MagicPlayer类中添加新方法：

```python
def new_spell(self, player, cost, groups):
    if player.energy >= cost:
        player.energy -= cost
        self.sounds['new_spell'].play()
        # 实现魔法效果逻辑
        # 例如：在前方创建粒子、造成范围伤害等
```

### 步骤4：在CombatManager.create_magic中集成

```python
def create_magic(self, style, strength, cost):
    if style == 'heal':
        self.level.magic_player.heal(...)
    if style == 'flame':
        self.level.magic_player.flame(...)
    if style == 'new_spell':
        self.level.magic_player.new_spell(...)
```

### 步骤5：测试验证

1. 按E键切换到新魔法
2. 验证能量消耗正确
3. 验证魔法效果正确
4. 验证无敌模式下伤害×2

## 添加新地图

### 步骤1：创建CSV文件

在 `data/map/` 目录下创建4个CSV文件：
- map_{id}_FloorBlocks.csv（边界/碰撞数据）
- map_{id}_Grass.csv（草地数据）
- map_{id}_Objects.csv（装饰物体数据）
- map_{id}_Entities.csv（实体数据）

### 步骤2：准备地面贴图

- `graphics/tilemap/ground.png`：地面贴图（用于水域检测）

### 步骤3：配置传送点

在Entities CSV中设置传送点编码，配置目标地图映射。

### 步骤4：测试验证

1. 通过传送点进入新地图
2. 验证地图正确渲染（四层叠加）
3. 验证碰撞检测正确
4. 验证水域检测正确
5. 验证玩家状态跨地图保留

BOSS_PHASE_DATA = {

def _on_boss_killed(self, death_pos):
    self.boss_kills += 1
    if self.boss_kills >= 4:  # 修改触发胜利的条件
        self._trigger_victory = True
        return
    self.spawn_boss(phase=self.boss_kills + 1)

4. 验证胜利触发条件正确

## 注意事项

1. **数据驱动原则**：新增内容优先通过配置实现，避免修改核心逻辑代码
2. **资源路径**：所有路径使用相对路径（相对于code/目录的 `../` 前缀）
3. **异常降级**：新资源加载失败不应导致崩溃
4. **性能影响**：新增大量精灵时注意帧率，必要时优化渲染逻辑
5. **向后兼容**：修改现有功能时确保不影响已有功能
