## 使用流程

### 开发前准备

1. **阅读系统提示词**：将 `00-项目系统提示词.md` 发送给AI，让AI理解项目规范
2. **阅读设计文档**：将需求规格说明书和概要设计说明书的关键章节发送给AI

### 分阶段开发

#### 第一阶段（核心跑通）

1. 将 `01-第一阶段-核心跑通.md` 发送给AI
2. AI逐个模块实现（settings→constants→support→entity→player→main→level→map_manager→pathfinding）
3. 每完成一个模块，AI会报告：职责、接口、依赖、测试点
4. 运行游戏验证验收标准
5. 使用 `04-代码审查与质量检查.md` 进行代码审查
6. 修复审查发现的问题

#### 第二阶段（战斗闭环）

1. 将 `02-第二阶段-战斗闭环.md` 发送给AI
2. AI实现：CombatManager→Weapon→MagicPlayer→Enemy→Particles→BossManager→BGM切换→无敌模式
3. 运行游戏验证验收标准
4. 代码审查 + 修复

#### 第三阶段（系统完善）

1. 将 `03-第三阶段-系统完善.md` 发送给AI
2. AI实现：UI→Upgrade→SaveManager→MapRenderer→场景切换→结算界面→水面检测
3. 运行游戏验证验收标准
4. 代码审查 + 修复

### 遇到问题时

1. 将 `05-调试与问题排查.md` 连同错误信息发送给AI
2. AI会分析根本原因并给出修复方案
3. 常见问题参考已在排查指南中列出

### 扩展新功能时

1. 参考 `06-扩展新功能指南.md`
2. 按步骤添加配置、资源、逻辑
3. 测试验证

### 文档更新时

1. 将 `07-文档生成提示词.md` 发送给AI
2. AI会根据代码状态生成/更新文档

## 提示词使用技巧

### 1. 上下文管理

- **不要一次性发送所有文档**：AI的上下文窗口有限，分阶段、分模块发送效果更好
- **先发系统提示词**：让AI理解项目规范后再开始编码
- **每次只聚焦一个任务**：避免AI同时处理多个复杂任务

### 2. 精准描述需求

- **明确输入输出**：告诉AI输入什么、输出什么
- **提供代码示例**：如果有参考代码，直接提供
- **指定文件路径**：明确告诉AI要修改哪个文件

### 3. 验证与反馈

- **每完成一个模块就测试**：不要等全部完成再测试
- **提供错误信息**：遇到问题时提供完整的报错堆栈
- **描述期望行为**：告诉AI应该发生什么，而不是只说"不对"

### 4. 代码审查

- **每个阶段结束后审查**：使用04-审查清单
- **关注架构合规性**：分层、单一职责、回调通信
- **关注性能**：帧率、内存、粒子系统

## 注意事项

1. **严格遵循设计文档**：AI的所有实现都应以设计文档为基准
2. **不要擅自发挥**：如果AI觉得哪里可以优化，先说出来讨论
3. **保持代码一致性**：命名规范、注释密度、文件结构保持一致
4. **测试驱动**：每个功能完成后立即测试验证
5. **文档同步**：功能变更后同步更新文档
