\section{项目背景与目标}

\subsection{【项目背景】}

我们几个组员都玩过《星露谷物语》，特别喜欢里面下矿洞打怪的部分——在一个俯视的小地图里，拿着不同的武器打各种小怪物，感觉又紧张又好玩。但星露谷的核心玩法是种田社交，战斗只是一小块内容，玩起来总觉得不过瘾。

我们又想到了《塞尔达传说》系列，它的战斗系统特别丰富——好多把武器可以随时切换，怪物也不是站桩挨打的，它们会追着你跑、会躲障碍物，打起来很有挑战性。

所以我们就想：能不能把星露谷的像素画风和塞尔达的战斗玩法结合起来，做一个\textbf{以打怪和探索为核心的小游戏}？让玩家在一个像素风的小世界里，拿不同的武器和魔法，跟各种智能怪物战斗，还能升级变强、探索更多地图。这个小游戏就叫「星露谷战斗模块」。

\subsection{【项目目标】}

我们讨论后确定了以下核心目标：

\textbf{必须实现的功能：}
\begin{enumerate}[leftmargin=2em]
    \item \textbf{好看的画面}：像素风格的俯视角画面，多个图层叠在一起（地面、草地、装饰、障碍物），画面下方的角色能挡住画面上方的角色，看起来有立体感
    \item \textbf{流畅的操控}：玩家可以用方向键控制角色上下左右移动，角色会根据移动方向切换动画（走路、站着不动、攻击三种状态，四个方向各一组，一共12组动画）
    \item \textbf{爽快的战斗}：按空格键攻击，根据角色朝向在对应方向生成武器，武器碰到敌人就扣血，不同武器攻击速度不一样
    \item \textbf{魔法技能}：两种魔法——治疗术回血、火焰术向前方扔火球范围伤害，释放魔法消耗能量值
    \item \textbf{武器切换}：按Q键在五把武器之间来回切换，每把武器伤害和攻速不同
    \item \textbf{聪明的怪物}：四种小怪物各有特色，它们会在原地发呆、看到玩家后追过来、靠近了就攻击，追的时候还会自动绕开障碍物
    \item \textbf{多张地图}：至少4张不同的地图，地图之间有传送门，走进传送门会淡出再淡入切换到新地图
    \item \textbf{升级变强}：打怪获得经验值，攒够了可以升级，把点数分配到生命、攻击等属性上
    \item \textbf{保存进度}：随时可以保存游戏，下次打开能继续玩
    \item \textbf{酷炫特效}：打中怪物有火花、怪物死了有爆炸特效、回血有光环、草被砍了有叶子碎片飞
\end{enumerate}

\textbf{基本要求：}
\begin{itemize}[leftmargin=2em]
    \item 游戏画面流畅不卡顿，每秒至少60帧
    \item 窗口大小1280$\times$720像素
    \item 代码结构清晰，分成多个文件，用面向对象的方式组织
    \item 资源文件用相对路径，换个电脑也能直接运行
\end{itemize}

\subsection{场景设计——游戏里有哪些画面}

\subsubsection{场景一：主菜单}

玩家打开游戏看到的第一个画面，我们要做得好看又有氛围：

\begin{itemize}[leftmargin=2em]
    \item \textbf{背景}：深蓝色渐变，营造一种神秘感
    \item \textbf{粒子效果}：蓝色的小光点慢慢往上飘，增加动感
    \item \textbf{标题}：金色大字「星露谷战斗模块」，像波浪一样上下浮动
    \item \textbf{副标题}：灰色小字，说明这是什么类型的游戏
    \item \textbf{菜单}：「开始游戏」和「退出游戏」两个选项，选中的会亮一下亮一下地闪烁
    \item \textbf{操作提示}：底部显示基本操作说明
    \item \textbf{音乐}：背景音乐一直循环播放
\end{itemize}

\subsubsection{场景二：游戏关卡}

这是游戏的核心部分，包含很多子系统：

\begin{itemize}[leftmargin=2em]
    \item \textbf{地图}：地面贴图打底，四层数据叠在一起渲染
    \item \textbf{镜头}：屏幕只有1280$\times$720，地图比屏幕大，镜头跟着玩家走
    \item \textbf{玩家角色}：像素小人，12组动画（上/下/左/右 $\times$ 走路/站着/攻击）
    \item \textbf{怪物}：四种小怪在地图上按预设位置出现，AI控制它们的行为
    \item \textbf{可破坏物}：草丛可以被攻击打掉，碎成叶子飞走
    \item \textbf{障碍物}：树、石墙等不能走过去的地方
    \item \textbf{传送点}：地图上特殊位置，走到上面就切地图
    \item \textbf{HUD}：屏幕顶上固定显示血条、能量条、武器、经验
    \item \textbf{升级菜单}：暂停后覆盖显示，半透明背景
\end{itemize}

\subsubsection{场景三：死亡界面}

\begin{itemize}[leftmargin=2em]
    \item \textbf{背景}：暗红色，营造一种「挂了」的氛围
    \item \textbf{标题}：红色大字「You Died」
    \item \textbf{统计}：显示这局打了多少经验
    \item \textbf{选项}：「按R重新开始」「按ESC退出」
    \item \textbf{粒子}：红色浮动粒子
\end{itemize}

\subsubsection{场景四：存档对话框}

\begin{itemize}[leftmargin=2em]
    \item \textbf{什么时候弹}：游戏启动时检测到有存档文件
    \item \textbf{长什么样}：圆角方框，深色背景金色边框，带阴影
    \item \textbf{内容}：标题「检测到存档！」，提示「按C继续 / 按N新游戏」
    \item \textbf{按钮}：[C] Continue（绿色）/ [N] New Game（红色）

\textbf{怪物的行为规则}（我们讨论了很久才定下来）：

怪物不是傻站着让你打的，它们有自己的「大脑」：

\begin{enumerate}[leftmargin=2em]
    \item \textbf{发呆状态}：玩家离得远的时候，怪物就在原地待着不动，不浪费计算资源
    \item \textbf{追击状态}：玩家走近了（进入警戒范围），怪物开始追玩家。追的时候不是直线冲过来——它会自己想办法绕过树啊墙啊这些障碍物，找到一条路追到玩家身边。每隔一段时间或者玩家跑到新的位置，怪物会重新想一条路
    \item \textbf{攻击状态}：追到玩家身边了（进入攻击范围），怪物就开始打玩家，打一下有短暂冷却
    \item \textbf{受伤反应}：被打中后有几百毫秒的无敌时间，身上会闪一闪让玩家知道
    \item \textbf{死亡}：血量降到0，播放死亡特效，玩家获得经验值，经验值会飞向玩家
\end{enumerate}

\subsection{怪物智能寻路——怎么让怪物追到玩家}

怪物追玩家的时候，不能直直地冲过来——地图上有树、有墙、有各种障碍物，怪物得自己想办法绕过去。这个问题有好几种解决思路，我们逐个讨论了：

\subsubsection{思路一：一层一层往外扩}

想象你站在地图中间，先看你周围一格能到哪些地方，再看两格能到哪些地方，一层一层往外扩，直到找到玩家。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：一定能找到最近的路，做法简单好理解
    \item \textbf{缺点}：太笨了——它会往所有方向都看一遍，不管玩家在哪个方向，浪费很多计算
    \item \textbf{适合场景}：地图很小、怪物很少的情况
\end{itemize}

\subsubsection{思路二：每次都走当前最短的路}

从怪物位置开始，每一步都选「到目前为止最近」的那个格子走，走到新格子后再更新所有格子的距离。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：一定能找到最近的路，还能处理不同地形走路代价不同的情况
    \item \textbf{缺点}：还是没有方向感，往所有方向都探，搜索范围比较大
    \item \textbf{适合场景}：地图有不同地形（草地走得快、泥地走得慢）的情况
\end{itemize}

\subsubsection{思路三：又快又准地找路（带方向感的搜索）}

每走一步都算一下「如果直接往玩家方向走还要走多远」，然后优先往「总距离最短」的方向走。这样既有方向感，又能保证找到最近的路。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：又快又准——既有方向感（知道往哪边走更近），又能保证找到最近的路
    \item \textbf{缺点}：需要想一个「估算距离」的公式，但一般不难
    \item \textbf{适合场景}：需要又快又准地找路的场景，比如实时游戏中怪物追玩家
\end{itemize}

\subsubsection{思路四：只看方向不看距离}

每一步都选「离玩家直线距离最近」的格子走，完全不管自己绕了多远。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：速度极快，直奔目标方向
    \item \textbf{缺点}：可能绕远路——比如前面有个墙挡着，它会贴着墙绕一大圈才到，而不是找一条更短的路
    \item \textbf{适合场景}：对路线质量要求不高、只求快速到达的场景
\end{itemize}

\subsubsection{思路五：提前算好全图路线}

先把地图上每个格子到目标的最短路都算好存起来，之后不管怪物从哪出发，查一下就知道怎么走。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：算一次之后查路线极快，怪物再多也不怕
    \item \textbf{缺点}：地图变了（比如草被砍了）就得重新算一遍全图；内存开销大
    \item \textbf{适合场景}：地图完全不变的情况
\end{itemize}

\subsubsection{思路六：多怪物协同规划}

不只考虑一个怪物怎么走，而是让所有怪物一起规划路线，保证它们不撞到一起。

\begin{itemize}[leftmargin=2em]
    \item \textbf{优点}：理论上能得到所有怪物的最优路线
    \item \textbf{缺点}：计算量太大了——怪物数量一多，计算时间呈爆炸式增长，根本算不过来
    \item \textbf{适合场景}：需要精确协调大量单位的策略游戏
\end{itemize}

\subsubsection{综合对比}

\subsubsection{我们的选择}

综合考虑下来，\textbf{第三种思路（又快又准）}最适合我们这个项目：它既能保证找到最近的路，搜索速度又快，还能应对地图变化（比如草被砍了路就通了）。

不过我们最终到底选哪种，还得等实际做出来测试一下性能再说。如果怪物太多导致卡顿，可以先用第四种（只看方向）作为备选方案。

\subsection{核心流程——游戏是怎么跑起来的}

\subsubsection{游戏整体流程}

\begin{enumerate}[leftmargin=2em]
    \item 游戏启动 → 显示主菜单
    \item 点「开始游戏」→ 加载地图和所有角色
    \item 每一帧循环：接收玩家操作 → 更新所有角色位置 → 检查碰撞 → 画画面 → 刷新屏幕
    \item 检查特殊事件（死了/走到传送点）→ 切换对应状态
\end{enumerate}

\subsubsection{打怪的流程}

\begin{enumerate}[leftmargin=2em]
    \item 玩家按空格 → 角色做出攻击动作
    \item 在角色朝向方向生成一个武器精灵
    \item 每一帧检查武器有没有碰到怪物
    \item 碰到了 → 怪物掉血 → 播放特效和音效
    \item 攻击冷却结束 → 武器消失 → 角色恢复正常
\end{enumerate}

\subsubsection{切地图的流程}

\begin{enumerate}[leftmargin=2em]
    \item 每一帧检查玩家是不是走到了传送点
    \item 走到了 → 画面慢慢变黑（淡出）
    \item 把当前地图的所有东西清理掉
    \item 加载新地图，创建新关卡
    \item 把玩家放到新地图的出生点
    \item 画面慢慢变亮（淡入）
    \item 玩家的血量、装备、经验都保留
\end{enumerate}

\section{总结与计划}

\subsection{讨论总结}

经过这次讨论，我们把「星露谷战斗模块」的需求和技术方案基本定了下来。简单说就是：做一个像素风的俯视角打怪小游戏，玩家能拿五种武器和两种魔法，跟四种智能怪物战斗，能升级、能换地图、能存档。

技术上用Python + Pygame来做，代码用面向对象的方式组织，地图用格子数据，存档用JSON格式，怪物寻路的方案等实际做的时候再测性能决定。
